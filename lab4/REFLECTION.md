This lab was challenging because understanding how processes communicate using pipes and how signals interrupt program flow required careful thinking about how the operating system manages execution. The hardest concept for me was handling synchronization between processes, especially understanding why closing unused pipe ends is necessary and how missing it can cause programs to hang. What clicked unexpectedly was how the shell actually works internally; realizing that commands like "ls | wc -l" are just fork, pipe, and exec made everything much clearer. Later, I would like to attempt building a more advanced shell with multiple pipes, job control, and possibly a history feature, as it would help me understand real-world terminal behavior more deeply.
